Как создавать релизы для updater:

Github Release

1. Структура релиза
Tag name: v1.0.0 (обязательно с префиксом v)

Release title: Version 1.0.0

Assets (файлы): должны называться в формате:

fear-windows-x86_64.exe (Windows 64-bit)

fear-windows-x86.exe (Windows 32-bit)

fear-linux-x86_64 (Linux 64-bit)

fear-linux-arm64 (Linux ARM64)

2. Пример правильного релиза
text
Release v1.2.3
├── fear-windows-x86_64.exe
├── fear-windows-x86.exe
└── fear-linux-x86_64

3. Как создать релиз:
Перейдите в репозиторий → Releases → "Draft a new release"

Tag version: v1.0.0

Release title: Version 1.0.0

Загрузите файлы с правильными именами

Опубликуйте релиз
